The Three Issues

Issue 1: Excessive vertical gap — brandingContainer has paddingVertical: 32px, creating ~64px of dead space between the SecurityStatusCard and the AI branding. The card already has its own bottom spacing.
Issue 2: Keyboard doesn't open on tap — The TextInput sits inside a KeyboardAvoidingView with SafeAreaView edges={['top','bottom']}. In the simulator, the software keyboard may be off by default (Cmd+K to toggle), but the real issue is that the welcome ScrollView-like content (the FlatList-less welcome state) doesn't properly integrate with keyboard avoidance. The input also lacks autoFocus or a tap-to-focus handler.
Issue 3: Suggestions clipped by tab bar — The AI screen lives inside MoreStack (a tab stack), so the bottom tab bar is always visible. The welcome container uses flex: 1 but there's no scroll — content that overflows the visible area gets clipped behind the tab bar and the input area.
