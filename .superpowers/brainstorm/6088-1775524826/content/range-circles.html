How should detection ranges appear on the map?

Each deployed device detects Bluetooth (~100ft), WiFi (~300ft), and Cellular (~800ft) signals. These ranges need to be visible on the Radar map so users understand their coverage.

Cellular ~800ft
WiFi ~300ft
Bluetooth ~100ft

A) Filled Concentric Circles ⭐ Recommended

Three semi-transparent filled circles centered on the device, matching the DeviceDetail Location tab style. Legend in the corner. Familiar pattern — same visual language already used in the app.

Pros

  • Consistent with DeviceDetail Location tab
  • Immediately clear coverage area
  • Works on both satellite and dark map styles

Cons

  • Large cellular circle can dominate the view at close zoom

B) Stroke-Only Rings

Dashed outline rings with no fill. Labels float on the ring edges. Cleaner look — doesn't obscure the satellite imagery underneath. More "tactical" feel.

Pros

  • Doesn't obscure satellite map detail
  • Cleaner, more tactical aesthetic
  • Labels right on the rings

Cons

  • Harder to see coverage "area" at a glance
  • Dashed lines can look busy on detailed maps
Ranges

C) Gradient Fill + Toggle

Subtle gradient fills (outer = faintest, inner = most visible) with a toggle button to show/hide ranges. Best of both worlds — coverage area is visible but can be dismissed.

Pros

  • User controls visual clutter
  • Gradient conveys signal attenuation
  • Toggle remembered per session

Cons

  • Extra UI element (toggle button)
  • More implementation work
